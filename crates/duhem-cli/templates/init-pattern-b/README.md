# `{{NAME}}`

A Duhem Verification Definition skeleton generated by
`duhem init --pattern B`. Sits as one feature directory under a
root manifest (`../duhem.yml`) per `docs/duhem-spec.md` §10.4.

- Criteria prose: [`criteria.md`](criteria.md)
- Verification Definition: [`duhem.yml`](duhem.yml)
- Sibling root manifest: [`../duhem.yml`](../duhem.yml) *(stub
  pending the manifest-loader spec)*

## What to do next

1. Edit `criteria.md` to express the real commitments of your
   feature.
2. Edit `duhem.yml` to mechanically verify each criterion.
3. Re-validate and re-run after each edit (commands below).

## Required inputs

| Input      | Type   | Default               |
| ---------- | ------ | --------------------- |
| `base_url` | string | `https://example.com` |

## Environment prerequisites

- Outbound HTTPS to the URLs `duhem.yml` targets.
- Playwright Chromium installed:
  ```sh
  npx playwright install chromium
  ```

## Running the verification

```sh
duhem validate {{PATH}}/duhem.yml
duhem run      {{PATH}}/duhem.yml
```

Override the default input:

```sh
duhem run {{PATH}}/duhem.yml --inputs base_url=https://example.org
```

## Root manifest note
